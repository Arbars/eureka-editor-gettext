
CHANGES IN Eureka 1.XX
======================

(Since version 1.11)


-  new 'File/Copy Map' command

-  LineDef panel : entering a negative "Length" moves the start vertex
   instead of the end vertex

-  Browser : changing categories clears the search box

-  undo, redo : show what was (un)done in the status area

-  find/replace : added a "Restrict to Selection" filter

-  find/replace : better way to choose textures/things

+  support for TX_START/TX_END textures, including PNG and TGA format

-  edit menu "Move/Scale/Rotate Objects" commands can be bound to keys

-  ability to skew (shear) objects with the mouse

-  a better dialog for the "JumpToObject" command

+  improved drag behavior, never modify the current selection

-  increased number of remembered recent files

-  the "Open Map" dialog was improved

-  find sprites outside of S_START..S_END (except in vanilla)

-  better default directory for file open/save dialogs

-  default edit mode is Vertices, and grid-snapping OFF


Key System

+  mouse buttons (and wheel) use the key binding system

-  line drawing is now done with RMB (MOUSE3) button

-  smooth scrolling/navigation in both 2D and 3D views

-  improved key binding dialog in the Preferences

-  allow keys hard-coded in the menus to be bound to other functions

-  changed scaling and rotation to the 'r' key (was MOUSE2)

-  changed map panning to use MMB (MOUSE2) or the 'a' key

-  changed grid toggle to the 'g' key


Game Defs:

+  properly detect which ports can be used with each game

+  support for STRIFE (based on the Slade3 config)

-  support for HARMONY

-  added "MBF" as a separate port

-  beginnings of a port definition file for ZDoom


Bugs Fixed:

-  fixed unwanted rail texture when splitting a sector and the
   texture name began with '_' or '#' (such as "_WALL").

-  fixed panel staying on a highlighted object when going to 3D mode

-  fixed problem of unable to split lines close to an end-point

-  fixed textures in LineDef panel not updating after undo or redo

-  fixed Export to adopt the iwad/port/resources of target wad

-  fixed rendering of unpegged lowers under sky ceilings (Y offset)

-  fixed "FIX" button for unknown textures to not change "-" textures

-  fixed how deleting a loop of linedefs inside another sector
   could sometimes delete that outer sector too
